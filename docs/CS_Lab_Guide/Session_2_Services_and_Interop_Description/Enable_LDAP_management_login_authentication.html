Enable LDAP management login authentication

Overview

Use section #2 and section #9 of the lab sheet.

Adding additional administrators to Pexip Infinity requires integration with an LDAP directory service, allowing these users to log in with their directory credentials, generally a Windows Active Directory domain. Infinity only has a single local administrator account, so integration with LDAP allows multiple users to administer the platform, allowing better auditing of changes and more control. You can assign specific groups of users' different privileges.

You need to complete three sections to set up LDAP for administrator authentication:

  • Set the actual connection to the LDAP server: This includes the paths and filters used to search the LDAP database for users looking to authenticate and the security groups used to assign specific privileges.
  • Set the administrator roles for Pexip Infinity: Two roles are defined out of the box (full admin read/write and a read-only role), although you can define custom roles depending on your needs.
  • Map the administrator role to a security group: Users within specific security groups can log into the Infinity system with a given privilege.

The local admin account login can be disabled for web access by ensuring the authentication source is the LDAP database only for more secure systems. If you have set this, then be aware that if Pexip Infinity cannot reach the LDAP database, web admin logins will NOT work. Resetting the authentication source can then only be reset via the SSH admin login. Currently, only one admin account is available to log in using SSH or the console, and you cannot add additional accounts.

It is also possible to force administrators to authenticate using client certificates as an additional security measure. These client certificates would typically be distributed via group policy (or similar). However, it is not something you will be using for this lab.

 

Procedure

First, you will need to set the LDAP server (the Domain Controller in your lab) and connection credentials that will bind to the LDAP database (the latter usually being some service account). Next, you will set the base Distinguished Name (DN) to form the starting point of any query against the LDAP service. Finally, you will append additional paths to narrow the search to a specific Organisational Unit (OU) for group and user lookup. You will usually use additional filters if you have an extensive directory structure.

 Remember, you also have access to the Domain Controller so that you can look at the Active Directory User's and Computer's snapin for more details.

You will use the default administrator roles and map them to the predefined security groups.

Once you have enabled LDAP in your lab, you should use the user account with read/write privileges to administer the system from now on. Again, check out your deployment against our screenshots to ensure you get the same results.

Please refer to https://docs.pexip.com/admin/managing_users.htm. In addition, LDAP syntax can be quirky, so referring to our LDAP troubleshooting guide can be a lifesaver (see https://docs.pexip.com/admin/integrate_ldap_troubleshooting.htm).

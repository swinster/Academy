Create a Virtual Meeting Room (Additional Configurations)

Use section #4.2 of the lab sheet.

In this step, you will create more VMRs and re-test the conferences. Again, test with one call placed internally to an internal node, and another externally to the edge node. Remember to check out Live View to see how things are different this time around.

Please also refer to https://docs.pexip.com/admin/about_aliases.htm, which outline many of the ways Pexip can match incoming calls.

 

Create five different VMRs with different options.

  • Create a secure VMR to allow only users with the correct PINs to enter the VMR.
    • Place an internal call using the host PIN, and an external call using the guest PIN.
    • See what the differences are for each users' in-conference capabilities.
  • Create a semi-secure VMR allowing easy access for guests.
    • The first user (guest) joins the conference, enters the lobby, and waits.
    • The next user (host) connects to the VMR, enters the lobby and enters the host pin to start the meeting. The guest has moved automatically from the lobby into the VMR.
  • Create a secure VMR only for C level users.
    • One user enters the host pin, and then the following user also enters the host pin. There is no option for guests to enter a pin or "join" the conference.
    • Both users are now connected to the VMR with the same rights.
  • Create a VMR to show two main speakers but do not allow presentation.
    • When the two users are connected to the VMR you will see a different layout.
    • Add more friends (PexEps) to test the layout with more people.
    • Now try to share your screen.
  • Create a VMR to show participant names and limit the room capacity to 3.
    • When the two users are connected to the VMR, the participants' names are displayed below each video feed.
    • Try to bring in your PexEp friends from earlier.

 

Make sure to test different layouts in different VMRs (such as Adaptive Composition). For example, you can set the default layout by configuring the "View" option. You select a layout per VMR service, but remember that all users receive the same layout.

NOTE: in a later task, we will customise the WebApp branding and add a plugin that allows a host to change this default layout from the controls in the client.

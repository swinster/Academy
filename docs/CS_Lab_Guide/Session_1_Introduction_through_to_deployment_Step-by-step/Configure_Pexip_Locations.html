Configure Pexip Locations

Suggested completion time: 15 minutes

More information

Use section #4.1 and section #5of the lab sheet.

You might wish to re-watch the video regarding System Locations.

Step-by-step

  1. Browse to the Management Node URL using its DNS name or IP address (accept the security warning if you see one, we will fix this later).
  2. Log in with username admin and the Pexip123! password.
  3. Go to Platform --> Locations and click the Add System Location button.
  4. Create a System Location for the internal nodes with the name LAN (see section #4.1).
  5. Configure DNS and NTP Servers as per section #4.1. If you need to add additional servers, click the green plus to add this configuration on the fly.
  6. Most other settings can be left as defaults. Note that the Transcoding Location is set to This location. Also note that this is where you would configure DSCP (QoS) settings and MTU size, meaning that all nodes within this location will inherit these settings.
  7. Save.
  8. Repeat to create another System Location for the edge nodes using the DMZ name (see section #5) and keep Transcoding Location: This Location (Note: Yes, this is initially going to be a Transcoding location!! We will change this later. Add public DNS and NTP Servers (Note: look for the green add button by the side of property boxes that allow you to add configurations on the fly. You could also navigate to System --> DNS Servers and add servers there).
  9. Save.

    NOTE: a warning will tell you that "No transcoding resource available" is available in each location. We will fix that in the next step.

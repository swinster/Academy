Understanding Infinity's IP networking and routing requirements is crucial to properly deploying an Infinity system, especially in more complex scenarios such as Cloud Bursting and the Pexip Private Cloud. You looked at some of this information in Level 1 eLearning, so let's recap that video.

 

To configure PSS:

Enable PSS:

  1. Go to Platform --> Global Setting --> Pexip Private Cloud.
  2. Tick Enable Pexip Private Cloud.
  3. Set the Gateway URL.
  4. Set the Customer ID.
  5. Set the Authentication token.
  6. Save.
  7. Ensure the Management Node icon is green () and there are no alarms raised.

 

Create a System Location to contain the PSS nodes:

  1. Go to Platform --> Locations.
  2. Add a new System Location.
  3. Set Name.
  4. Set DNS and NTP Servers (remembering that external servers may not be reachable).
  5. Set MTU.
  6. Transcoding Location: This location.
  7. Save

 

Configure PSS location:

  1. Go to Platform --> Pexip Smart Scale
  2. Set Region.
  3. Set Max HD Calls
  4. Set System Location.
  5. Save.

 

Enable PSS location:

  1. Check the box next to the location, then in the Action drop-down, select Enable then click Go.

Configure the DMZ System location:

  1. Platform --> Locations, then select the DMZ location.
  2. Set Primary overflow location as outlined in the lab sheet.
  3. Set the MTU.
  4. Then Save.

 

Configure the DMZ Node:

  1. Platform --> Conferencing Nodes, then select the Edge node.
  2. In the Static routes section, click the green plus to Add another static route.
  3. Configure the static route, as per the details in section #14 of the lab sheet.
  4. Click Save to store the static route configuration.
  5. Click Save again to update the Conferencing Node.

 

Configure the LAN System location:

  1. Platform --> Locations, then select the LAN location.
  2. Set Primary overflow location as outlined in the lab sheet.
  3. Set the MTU.
  4. Then Save.

 

Configure the Management Node:

  1. Platform --> Management Nodes, then select the Management Node.
  2. Set the MTU.
  3. Then Save.

 

Testing

Once PSS has been configured and fully deployed, place internal and external calls into your Infinity deployment. You may need to add some PexEP friends to add additional load. Initially, you should see that the transcoding resources in the LAN are used. Infinity will start to overflow media to the PSS location when the internal transcoding resources are full.

Here’s a quick question and answer for you:

Q. What are the consequences of setting up a transcoding location (such as the LAN) to use an overflow location?

A. Any endpoint connecting directly to a node in the LAN location will also need a direct route to the PSS location. Infinity will signal to the endpoint that it must send media directly to the PSS nodes if overflow happens. The Transcoding Nodes will NOT act as a Proxying Edge Node. Firewall rules may not allow this traffic. Instead, you may wish to reconfigure the Transcoding Nodes to use the Proxying Edge role and then define the transcoding resource of the LAN location to be that of the PSS location. Alternatively, you could add another internal System Location containing Proxying Edge Nodes.

Extra Credit: LDAP VMR Sync/import

Suggested completion time: 40 minutes

More information

Use section #10 and section #9 of the lab sheet.

Configuring the LDAP sync source:

  1. Go to Utilities --> LDAP sync source --> Add LDAP sync source.
  2. Give it a proper name, i.e. the name of your LDAP server.
  3. Set the LDAP server.
  4. Set the LDAP bind username.
  5. Set the LDAP bind password.
  6. Configure the LDAP base DN: dc=pexip,dc=net
  7. Save.

 

Configuring the LDAP sync template (to get the initial data set):

  1. Utilities --> LDAP sync template --> Add LDAP sync template.
  2. Name: Norway VMRs.
  3. LDAP user search DN: OU=NO,OU=DemoUsers
  4. Choose your Sync source.
  5. Sync VMRs.
  6. VMR name pattern: VMR Norway {{givenName}} {{sn}} ({{mail}})
  7. VMR description pattern:
    8. telephoneNumber: {{telephoneNumber}} - mobile: {{mobile}} - employeeID: {{employeeID}} - sAMAccountName: {{sAMAccountName}}
  8. Save.

    9. NOTE: you have not yet added VMR aliases. However, we have added some key data points into the VMR name and VMR description fields to show information that we can use to build these aliases later.

 

Perform the initial sync of data:

  1. Re-edit the sync template just created.
  2. Click the “Sync Now” button.

 

Check all the Sync templates on the Sync Template page, Sync and review the data. Based on the data you see in the description fields, choose a VMR scheme for alphanumeric and numeric aliases.

  1. Edit the sync template.
  2. Add the alias pattern:
    3. meet.{{mail|lower|replace("@pexip.net", "@<your_group>.pexip.net")}}

    Example: Prefix with meet, use variable mail, convert to lowercase, replace original domain in email with your Pexip-domain (also see https://docs.pexip.com/admin/sync_patterns.htm).

  3. Resync the template.

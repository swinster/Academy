Use section #5 of the lab sheet.

Refer to https://docs.pexip.com/admin/about_h323_sip.htm and https://docs.pexip.com/admin/integrate_pstn.htm.

Use the "General VR", which you configured in a previous lab, and extend its operation to handle PSTN calls. This Virtual Reception should currently have an alias of lobby@<your Pexip Infinity domain> (hopefully, you know where to go by now) and possibly the IP address of the edge node. To handle PSTN calls, you will need to add a new alias to the VR. This alias will need to match the PSTN number provided, which will allow calls to route to your VR (via a Twilio VoIP SIP trunk). Your trainer will provide you with the PSTN number for your location.

You can then try to dial into a VMR using a phone.

The destination VMR must have one alias that is purely numeric so that users can enter it as touch-tones (DTMF) via the phone's keypad. You can then dial into the VR and transfer by tapping the VMRs numeric alias. The Virtual Reception phone number and VMR conference code would typically be included in a calendar invite, for example, as provided by the Pexip Scheduling solution.

An example use case could be to provide many customised Virtual Receptions with different voice prompts in other languages that would allow users to connect using alternate aliases.

Edit and use a Test Call service

Use section #4.2 of the lab sheet.

The Pexip Infinity Test Call Service is a feature both administrators and end-users can use to test and validate connections from different devices and clients. There is already a test call service configured out of the box, which uses an alias of “test_call”. You can change this alias if you like, but you should also add a new alias that contains the Pexip Infinity Domain for your deployment (see section #4.2 in the lab sheet)

Multiple aliases can be configured for every Infinity service, allowing various clients to connect to the same service differently. For example, a SIP client would dial a URI using user@domain. Therefore, you would configure a suitable alias on the service to match this incoming call (using your Pexip Infinity Domain for your lab from section #4.2). However, you may only want WebRTC users to type the shorter user part to access the same service.

To test this service using webRTC, open a browser to an internal node (using a Jumpbox browser bookmark), and the edge node using its FQDN from your device. Note and accept the security warning in the browser. We will add TLS certificates to the Infinity nodes in the next lab. Also, note that the Jumpbox server has a virtual camera installed, allowing you to send a test video stream.

Check out the screenshots on the previous page to ensure your deployment is working as expected.

Create a Virtual Meeting Room (with no PIN)

Use section #4.2 of the lab sheet.

The second Infinity service we will look at is the Virtual Meeting Rooms (VMR). VMRs are the main meeting spaces in an Infinity deployment and have many configuration options. In the next lab, you will create different VMRs with different properties. For this step, however, you will create a standard VMR with no PIN codes that you can connect to using SIP and H.323 devices and a shorter name for WebRTC users. 

To achieve this, make sure you set a couple of aliases in a VMR named "General VMR":

  • The short alias for WebRTC users will be vmr.
  • The longer alias needs to include your Pexip Infinity Domain.

Please refer to https://docs.pexip.com/admin/creating_editing_vmrs.htm.

 

You have now created your first VMR. Before you can make a call to the VMR, the configuration must synchronise to the Conferencing Nodes, which can take up to 60 before they take effect. Every 60 seconds, each Conferencing Node checks with the Management Node to see if any configuration changes are available. When the database with your new VMR is replicated to the Conferencing Nodes, you will be able to dial in and connect.

NOTE: from v27 we have made some changes to how nodes get configuration updates. We will continue to improve in upcoming releases and speed up the synchronisation process.

You can place calls into your VMR using WebRTC via our browser. First, navigate to the Conferencing Node FQDNs. The nodes will have an untrusted self-signed certificate applied, so you must accept the browser warnings. You will fix the security issue in a later lab.

Place one call to the DMZ node (e.g. https://edge.<Pexip_Infinity_domain>) from a browser outside the Jumpbox session. Then place another call to an internal LAN node (e.g. https://conf02.pexipacademy.local), this time using a browser bookmark inside the Jumpbox session. Be aware that your web camera and microphone may not work via the Jumpbox session, however, there is a virtual camera installed on the Jumpbox server that will provide a video feed for testing.

Once you have completed this, you will automatically create your first distributed conference. Check out Live View to see what is happening (please refer to https://docs.pexip.com/admin/platform_status.htm). Note the backplane between the two nodes. Remember, at the moment, all nodes in your deployment are in the transcoding role. Media landing in the DMZ is terminated and decoded, then sent (if required) to other nodes in other locations where it is mixed, encoded and sent to other endpoints. The same process is happening on the nodes in the LAN location.

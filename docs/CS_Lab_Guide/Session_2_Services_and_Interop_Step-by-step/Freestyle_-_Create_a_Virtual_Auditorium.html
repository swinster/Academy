For this task, we are not going to outline any steps 😱. Instead, you should refer to https://docs.pexip.com/admin/creating_editing_lectures.htm to complete the task. The point of this exercise is that you will refer to our documentation.

So, go and create your Virtual Auditorium!

Most of the time you when you create a Virtual Auditorium, you will do so to allow both hosts and guests, so you will want to ensure that you have set a Host PIN and "Allow guests" is set to "Yes".

What we will say is that you should use the following information:

  • Auditorium name = Lecture Room
  • Host Pin = 5678
  • Aliases = lecture and lecture@<Pexip_Infinity_doamin>.

 

Testing

Create some calls into the Virtual Auditorium.

  • Create a call into an internal node using the browser within the Jumpbox session. Join as a host.
  • Then, create another call using the browser on your laptop to connect to the edge node. Join as a guest.

Perhaps a lecture is a little lonely with just two calls, so invite some friends in the form of PexEPs (Pexip EndPoints). The PexEPs are a separately deployable VM that generate video calls to help you test an Infinity system, and we have already set some up for you in the lab.

NOTE: When you get back to your day job in your organisation (i.e., outside of these labs), you can use our PexEPs to test your corporate Infinity deployments. If you want to set these up in your networks, you should ensure that you deploy the PexEP VM on your internal corporate network only. Be aware that these VMs use ancient code, and no security patches have been applied in a long time. In addition, they should NOT be publicly accessible as SPAMers could access them to generate calls. Check out the Additional tools resource in the course material for more information.

Use the following link to place calls into your LAN system location from the Jumpbox session only. Then, you can use the bookmarks in the browsers via the Jumpbox session:

  • http://pexep-int1.pexipacademy.local/ (SD)

For the Meeting room URI, add the URI alias you have just created for your VA. You can use just the user part of the hostname, e.g. lecture.

For the Conferencing Node IP or hostname, the host FQDN of one of your internal nodes is already populated (conf01.pexipacademy.local, also see Section #4).

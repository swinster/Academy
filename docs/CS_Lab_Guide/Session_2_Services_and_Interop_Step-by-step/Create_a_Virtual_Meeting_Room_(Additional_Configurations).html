Create a Virtual Meeting Room (Additional Configurations)

Suggested completion time: 30 minutes

More information

Use section #4.2 of the lab sheet.

Before configuring some of the Infinity services that handle conferences and other call types, it's good to recap what firewall configuration changes are required to allow external endpoints and systems to connect to Infinity.

 

Configure Virtual Meeting Rooms (VMRs) via the Services menu: Services --> Virtual Meeting Rooms.

 

Create a secure VMR to allow only users with the correct PINs to enter the VMR. You will enable both hosts and guests to enter; both require a PIN. Hosts can manage the conference whilst guests simply participate:

  1. Configure a name (Secure VMR) and description, set the host pin and the guest pin, and allow guests.
  2. Configure aliases like secure and secure@<Pexip Infinity domain> (see lab sheet section #4.2).

  3. Save and wait for sync before placing test calls to the VMR alias.

 

Create a semi-secure VMR allowing easy access for guests:

  1. Configure a name (Semi-Secure VMR) and description, set the host pin and allow guests to yes, with a blank guest pin.
  2. Configure aliases like semisecure and semisecure@<Pexip Infinity domain> (see lab sheet section #4.2).
  3. Save and wait for sync before placing test calls to the VMR alias.

 

Create a secure VMR only for C level users:

  1. Configure a name (C level VMR) and description, set the host pin and allow guests to no.
  2. Configure an alias like clevel and clevel@<Pexip Infinity domain> (see lab sheet section #4.2).
  3. Save and wait for sync before placing test calls to the VMR alias.

 

Create a VMR to show two main speakers but do not allow presentation:

  1. Configure a name (No Presentation VMR) and description, leave host and guest pin blank as default. Change the layout to 2+21.
  2. In the Advanced options section, then set the Conference capabilities to Main video only
  3. Configure an alias like nopresentation and nopresentation@<Pexip Infinity domain> (see lab sheet section #4.2).
  4. Save and wait for sync before placing test calls to the VMR alias.

 

Create a VMR to show participant names and limit the room capacity to 3:

  1. Configure a name (Limited Participant VMR) and description, leave the host and guest pin blank, and the other configuration as default, except Show names of participants as yes.
  2. In the Advanced options section, then set the Participant limit to 3.
  3. Configure an alias like smallvmr and smallvmr@<Pexip Infinity domain> (see lab sheet section #4.2).
  4. Save and wait for sync before placing test calls to the VMR alias.

 

To set the default layout on any of the VMRs to use Adaptive Composition, in the main VMR configuration section, set "View" to "Adaptive composition (automatic face detection and framing)". You should also review other available layouts.

 

Check the configuration of your Pexip Infinity VMR services

When you have created all the VMRs, test all the services. Be sure to connect multiple endpoints using the PexEPs, adding PINs if required. You might want to go beyond our steps and try out other options and layouts.

In Pexip Infinity, you can apply customisations to most user-facing components so that you can deliver something unique to you and your users and customers. For example, you can fully customise the Connect Client interface (including the web app, desktop and mobile), in addition to the in-call experience. Brands and Themes are different concepts that apply to the client user interface and the in-call experience.

As of v17, we introduced a new user experience (known as the NextGen Connect client), and the older version is now known as the Legacy client. In addition, we have created external web portal services to create branding templates for free. The NextGen portal can customise the entire User Experience (both client interface and in-call experience).

Both portals can be accessed via the landing page at https://brandingportal.pexip.com//, and each requires a separate login, so you will need to create an account for both. However, we will concentrate on the NextGen branding in this exercise only.

When creating custom branding, the web app experience is defined by a web template that you upload to the Management Node (or optionally to a reverse proxy). If you so wish, it can be further customised manually.

 

NextGen Portal

The portal experience blurs the lines between client user interface customisation (known as "branding") and the in-call experiences (known as "theming"). However, as these are applied to different elements of Pexip Infinity in different ways, we have continued to document these areas separately. See https://docs.pexip.com/admin/customize_clients.htm for information on customising the client experience, and https://docs.pexip.com/admin/themes.htm for information on customising the in-call experience.

Be aware of design choices for backgrounds where users will see messages (such as VR's and the "waiting for host" prompt). Please do not make them busy or distracting. Think about accessibility (i.e. colour clashes, colour blindness etc., see https://docs.pexip.com/admin/accessibility.htm), and logo watermarks are often better as an all-white, 40% transparent image. Some excellent royalty-free background images can be found at https://unsplash.com/

There are four main edit sections (subject to change). Go through each section and create a NexGen user experience.

  • App Edits: This allows you to change the look and feel of the Connect app and some in-call experience screens, plus adding the watermarking logo.
  • Customisation: Here, you can change how the Connect app functions, such as the bandwidth options that are exposed to the user.
  • Splash Screens: You can adjust the main in-call experience themes in this section.
  • Languages: Here, you can set up translation strings for each of the static text elements within the clients. These user-defined languages can then be assigned to the Connect client via the Customisation menu.

To create a branding/theme package, you can select one or several of your edits (App Edits, Customization and/or Splash Screens), which will then create one or two zip files:

  • The branding.zip is a combination of the App Edits, Customisation and Languages sections which is uploaded as the web app customisation (Service Configuration à Web App Customization).
  • And ivr.zip, mainly containing the Splash Screens edits and incorporates the watermark logo from the App Edits section, uploaded as a theme (Service Configuration --> Themes) and assigned to individuals Infinity services, or globally across the deployment.

Test out the new client interface and Infinity services.

NOTE: you can also brand the desktop client, although this requires more work. Rember, though, you should always consult our documentation here - https://docs.pexip.com/admin/customize_clients.htm#apply. From Infinity v25, you will need to ensure that the branding package you create is digitally signed, and the client must upload a trusted (public) key before branding can be applied. Unfortunately, version 1.8 on the Connect client cannot apply branding to mobile clients due to additional security restrictions.

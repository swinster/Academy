In Pexip Infinity you can apply customisations to most user-facing components so that you can deliver something unique to you, and for your users and customers. For example, you can fully customise the Connect Client interface (including the web app, desktop and mobile), in addition to the in-call experience. Brands and Themes are the different concepts applying to the client user interface and the in-call experience respectively.

As of v17, we introduced a new user experience (known as the NextGen Connect client), and the older version is now known as the Legacy client. We have created some external web portal services where you can create branding templates for free; the NextGen and Legacy portals, used for each client group respectively. The NextGen portal allows for the customisation of the entire User Experience (both client interface and in-call experience), whereas the Legacy portal only allows for web app and desktop client customisation.

Both portals can be accessed via the landing page at https://brandingportal.pexip.com/, and each requires a separate login, so you will need to create an account for both. However, in this exercise, we will concentrate on the NextGen branding only.

When you create custom branding, the web app experience is defined by a web template that you create and upload to the Management Node (or optionally to a reverse proxy). This can be further customised manually if you so wish.

NextGen Portal

The portal experience is blurring the lines between client user interface customisation (known as Branding), and the in-call experiences (known as Theming), however, as these are applied to different elements of Pexip Infinity and in different ways, we have continued to document these areas separately. See https://docs.pexip.com/admin/customize_clients.htm for information on customising the client experience, and https://docs.pexip.com/admin/themes.htm for information on customising the in-call experience. 

Be aware of design choices for backgrounds on the areas where users will see messages (such as VR’s and the “waiting for host” prompt), these should be not too busy or distracting. Think about accessibility (i.e. colour clashes, colour blindness etc, see https://docs.pexip.com/admin/accessibility.htm), and logo watermarks are often better as an all-white image. Some nice royalty-free background images can be found at https://unsplash.com/. 

There are four main edit sections (subject to change). Go through each section and create a NexGen user experience. 

  • App Edits: This allows you to change the look and feel of the Connect app and some in-call experience screens, plus adding the watermarking logo.
  • Customisation: Here you can change how the Connect app functions, such as the bandwidth options that are exposed to the user. 
  • Splash Screens: In this section, you can adjust the main in-call experience themes.
  • Languages: Here you can set up translation strings for each of the static text elements within the clients. These user-defined languages can then be assigned to the Connect client via the Customisation menu.

To create a branding/theme package, you can select one or several of your edits (App Edits, Customization and/or Splash Screens), which will then create one or two zip files: 

  • branding.zip is a combination of the App Edits, Customisation and Languages sections which is uploaded as the web app customisation (Service Configuration à Web App Customization).
  • and ivr.zip which is mainly the Splash Screens edits but also incorporates the watermark logo from the App Edits section, which is then uploaded as a theme (Service Configuration --> Themes) and assigned to individual Infinity services, or globally across the deployment. 

Ttest out the new client interface and Infinity services. 

NOTE: you can also brand the desktop client. This requires a bit more work, but as always, things are fully documented here - https://docs.pexip.com/admin/customize_clients.htm#apply. From Infinty v25, you will need to ensure that the branding package you create is signed, and the client must upload a trusted (public) key before the branding can be applied. Unfortunately, as of version 1.8 on the Connect client, you cannot apply branding to the mobile clients. This is due to additional security restrictions.

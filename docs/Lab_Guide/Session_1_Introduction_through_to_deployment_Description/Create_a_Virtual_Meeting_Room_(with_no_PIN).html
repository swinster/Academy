Create a Virtual Meeting Room (with no PIN)

Use section #4.2 of the lab sheet.

One of the services in Pexip Infinity is Virtual Meeting Rooms (VMRs). When you create Pexip VMRs, you have multiple configuration options you could set. In the upcoming lab exercises, you will create different sorts of VMRs with different properties. For this step, create a standard VMR with no PIN codes that you can connect to using SIP devices and a shorter name for WebRTC users. Set the user part of the alias to vmr.

Please refer to https://docs.pexip.com/admin/creating_editing_vmrs.htm.

 

You have now created your first VMR. Before you can make a call to the VMR, the configuration must synchronise to the Conferencing Nodes, so you must wait up to 60 seconds after making changes before they take effect. Every 60 seconds, each Conferencing Node checks with the Management Node to see if any configuration changes are available. When the database with your new VMR is replicated to the Conferencing Nodes, you will be able to dial in and connect.

To connect to your VMR, you can call from a browser. To do so, you can navigate to the Conferencing Node FQDNs, however for now the Nodes will have untrusted self-signed certificates, so you will need to accept the browser warnings (this will be fixed in a later step).

If one user in the group connects to the external node (e.g. https://edge.<your_Infinity_domain>) outside of the RDP session, and the other user connects to an internal node (e.g. https://conf02.<your_Infinity_domain>), you should be able to create a distributed conference. Be aware that your web camera and microphone may not work via the RDP session (there is a virtual camera installed on the RDP server, but it has limited functionality). The trainer should also be able to connect to an internal node with an internal client to help you out if required.

Check out Live View to see what is going on (please refer to https://docs.pexip.com/admin/platform_status.htm). Note the backplane that is created between the two nodes. 

To understand how Pexip handles a distributed conference, please read through the documentation at https://docs.pexip.com/admin/dist_architecture.htm. This might seem like a lot of reading, but to fully understand Pexip Infinity, you need to understand how it operates. Indeed, you might need to read these pages several times as Pexip’s architectural approach is unlike any other conferencing solution.

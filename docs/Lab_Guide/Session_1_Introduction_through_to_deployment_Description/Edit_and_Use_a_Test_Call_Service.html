Edit and Use a Test Call Service

Use section #4.2 of the lab sheet.

The Pexip Infinity Test Call Service is a feature both administrators and end-users can use to test and validate connections from different devices and clients. By default, there is already a test call service configured out of the box, which uses an alias of “test_call”. You can change this alias if you like, and you should also add a new alias that contains the Pexip domain for your deployment. 

Multiple aliases can be configured for Pexip Infinity services that allow different clients to connect to the same service. For example, a SIP client would dial a URI in the format user@domain, and you would need to configure a suitable alias on the service to match this incoming call (using the Pexip Infinity Domain for your group). However, you may only want WebRTC users to have to type the shorter user part to access the same service.

To test this service using webRTC, browse to an internal node and/or the edge node using its FQDN. Note the security warning in the browser. We will add TLS certificates to the Pexip Infinity nodes in the next session, so for the moment accept the warning and move on.

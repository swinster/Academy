Create a Virtual Meeting Room (with no PIN)

Suggested completion time: 15 minutes

More information

Use section #4.2 of the lab sheet.

  1. Navigate to Sevices --> Virtual Meeting Room, then click Add VMR.
  2. Select a name and description. The name and description are visible if a Skype client adds the VMR to the buddy list/roster. Construct your VMR name and description with this potential exposure in mind.
  3. Leave host pin blank, allow guests as no, and guest pin blank.
  4. Leave all other fields as default.
  5. On the alias section, create one alias called vmr@<Pexip Infinity domain> (see lab sheet section #4.2 to figure out your domain). Create another alias using only vmr and no domain.
  6. Click Save.
  7. Wait 1 minute for the changes to be replicated to all the nodes.

Screenshot 4: Demonstrate your working Pexip Infinity deployment

Each person in the group places a call into the VMR service. Ensure that at least one person calls into the service through the LAN location, and one person calls into the same service through the external location at the same time.

  • You can connect to an internal node using a browser within the RDP session and the URL of a LAN nodes FQDN or IP address (you will have to accept the warning for now), e.g. https://conf02.<your_Infinity_domain>
  • Connect to the DMZ node outside of the RDP session, from your laptop or mobile device, e.g. https://edge.<your_Infinity_domain>.

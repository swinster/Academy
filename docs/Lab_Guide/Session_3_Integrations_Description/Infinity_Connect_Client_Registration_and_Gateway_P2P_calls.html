Infinity Connect Client Registration and Gateway P2P calls

Use section #4.2 of the lab sheet.

Routing rules are one of the most flexible and complicated configuration pages in Pexip Infinity, so let’s practice and at the same time download, install and register the Pexip Connect client for Infinity. This client should be installed on YOUR computer, not the RDP server. In addition, the Infinity Connect client is not the same thing as the Pexip My Meeting Video client used by the Pexip Service. We are working on a replacement app that can be used by both Infinity and the Service, so watch this space.

We need routing rules to be added to enable registered devices to call each other, and both call and be called by, external systems. These are known as gateway calls.

Please refer to https://docs.pexip.com/admin/connect_installing.htm.

  1. Setup a Device Alias (this is the identifier used to dial the device – usually it will look like a URI), and register a Pexip Connect client to your deployment.

    For registering to a Pexip Conference or Proxy Node, you must first create a Device Alias. Refer to https://docs.pexip.com/admin/registering_devices.htm and https://docs.pexip.com/end_user/guide_for_admins/about_client_settings.htm

  2. Configure a Call Routing Rule so it is possible to dial from internal and external systems to registered clients @<Pexip Infinity domain>.

    The key here is to ensure your RegEx rule matches your domain, and that the call target is directed towards “Registered devices only”. You probably want to ensure that both “inbound” and “outbound” calls are matched, so you can call a registered device directly from a VMR.

  3. Configure a Call Routing Rule so you can dial from registered devices (and VMRs) to any public IP as H.323 – ideally, you can test with an available hardware endpoint (Note: you can get extra credit if you can explain how the IP match RegEx works!).

    This is similar to the above, although the RegEx match needs to capture a dialled IP address. There are several ways to do this, so check the RegEx examples to cut’n’paste.

  4. Configure a Call Routing Rule so you can dial from registered devices (and VMRs) to any external system via the DMZ (something@domain) as SIP – ideally, you can test with an available hardware endpoint.

    Here, it is important to ensure that you do not cause a loop. Ignore your domain (you can use a negative lookahead RegEx example) and that the inbound call is matched only from registered devices. Often, these are one of the last rules to add to a system.

 

Also refer to https://docs.pexip.com/admin/configuring_gateway_rules.htm, https://docs.pexip.com/admin/about_gateways.htm and https://docs.pexip.com/admin/regex_reference.htm.

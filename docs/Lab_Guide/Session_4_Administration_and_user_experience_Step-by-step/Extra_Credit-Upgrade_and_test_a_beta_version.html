Extra Credit: Upgrade and test a beta version

Suggested completion time: 20 minutes

Use section #3 of the lab sheet.

See the previous upgrade section for the procedure, but this time upgrade to a beta version of Infinity. Ensure you have backed up the configuration and created VM snapshots before you upgrade (especially given this is beta software), and remember to remove the VM snapshots when you have tested your system.

Beta software should only be run in test environments, as it will contain pre-release code that could have unintended side effects. It’s a great way to test new features or to ensure that any documented change in behaviour helps with a particular issue.

There are some rules to be aware of when upgrading:

  • The beta version you are upgrading to, MUST have been released AFTER the version of software you are running. Each new software version is coded with previously released versions, so you might find some beta software is older than the GA release of a previous version. Check the dates of release of the software.
  • You can only upgrade from a beta version to the next beta, release candidate, or GA version of the same release (i.e. v26 beta 2 will allow you to upgrade to v26 GA, but not v26.1 GA or v27 beta 1).
  • Released beta versions should be supported in the Pexip Private Cloud. If you see issues after upgrading relating to PSS not support the current version, please let your trainer know.

 

Ask your trainer for links to the beta documentation and information about the new version.

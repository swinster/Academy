Pexip Smart Scale

Suggested completion time: 20 minutes

More information

Use section #13 of the lab sheet.

To configure PSS:

Enable PSS:

  1. Go to Platform --> Global Setting --> Pexip Private Cloud.
  2. Tick “Enable Pexip Private Cloud”.
  3. Set the Gateway URL.
  4. Set the Customer ID.
  5. Set the Authentication token.
  6. Save.
  7. Ensure the Management Node icon is green () and there are no alarms raised.

 

Create a System Location to contain the PSS nodes:

  1. Go to Platform --> Locations.
  2. Add a new System Location.
  3. Set Name.
  4. Set DNS and NTP Servers (remembering that external servers may not be reachable).
  5. Set MTU.
  6. Transcoding Location: This location.
  7. Save

 

Configure PSS location:

  1. Go to Platform --> Pexip Smart Scale
  2. Set Region.
  3. Set Max HD Calls
  4. Set System Location.
  5. Save.

 

Enable PSS location:

  1. Check the box next to the location, then in the Action drop-down, select Enable then click Go.

Configure the DMZ System location:

  1. Platform --> Locations, then select the DMZ location.
  2. Set Primary overflow location as outlined in the lab sheet.
  3. Set the MTU.
  4. Then Save.

 

Configure the DMZ Node:

  1. Platform --> Conferencing Nodes, then select the Edge node.
  2. In the Static routes section, click the green plus to “Add another static route”. 
  3. Configure the static route, as per the details in section #13 of the lab sheet.
  4. Click Save to store the static route configuration.
  5. Click Save again to update the Conferencing Node.

 

Configure the LAN System location:

  1. Platform --> Locations, then select the LAN location.
  2. Set Primary overflow location as outlined in the lab sheet.
  3. Set the MTU.
  4. Then Save.

 

Configure the Management Node:

  1. Platform --> Management Nodes, then select the Management Node.
  2. Set the MTU.
  3. Then Save.

 

Testing

Once PSS has been configured and is fully deployed, place some calls from both internal and external endpoints. You may need to add some PexEP friends to add additional load. Initially, you should see that the internal LAN transcoding resources will be used. When the internal transcoding resources are full, Infinity will start to overflow media to the PSS location.

Here’s a quick question and answer for you:

Q. What are the consequences of a Transcoding location (such as the LAN) being set up with an overflow location?

A. Any endpoint connecting directly to a node in the LAN location will need to also have a direct route to the PSS location. If overflow happens, Infinity will signal to the endpoint that it must send media directly to the PSS nodes. The Transcoding nodes will NOT act as a Proxying Edge node. Firewall rules may not allow this traffic, so you may wish to reconfigure the LAN nodes to be in the Proxying Edge role, and then define the transcoding resource of the LAN location to be that of the PSS location, or add another internal System Location containing Proxying Edge nodes.
